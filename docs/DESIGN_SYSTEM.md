# HAPI Design System

## 1. Design direction

Premium, clean, nature-backed, science-aware.

The design should follow the visual reference created for HAPI:

- dark premium hero section;
- product-forward layout;
- soft neutral background;
- clear product cards;
- ingredient cards with line icons;
- lifestyle audience cards;
- strong AI consultant block.

## 2. Layout principles

- Large whitespace.
- Strong visual hierarchy.
- Product images should be prominent.
- Cards should be soft, rounded, and clean.
- Avoid clutter.
- Mobile-first responsive layout.

## 3. Core sections

### Hero

Dark background, nature texture, product image, strong headline.

Suggested headline:

> Real ingredients. Real impact.

CTA buttons:

- Explore products
- Find your HAPI

### Product cards

Each card includes:

- product name;
- use case;
- short copy;
- color-coded background;
- product visual;
- learn more link.

### Ingredient cards

Each card includes:

- icon;
- ingredient name;
- functional role;
- short explanation.

### Audience cards

Lifestyle image + audience label:

- Busy professionals
- Moms & parents
- Athletes & fitness lovers
- Travelers
- Drivers
- Students

### AI consultant block

Dark premium block.

Headline:

> Not sure which HAPI is for you?

CTA:

> Find my HAPI

## 4. Component structure

Use reusable components:

- Header
- Hero
- TrustStrip
- ProductCard
- ProductGrid
- IngredientCard
- AudienceCard
- AiConsultantBlock
- Newsletter
- Footer

## 5. Technology

Recommended:

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui if useful
- Vercel deployment

## 6. Code principles

- No hardcoded product chaos.
- Product information must live in structured data files.
- Components must be reusable.
- Pages must be clean and modular.
- All copy must be easy to update.
